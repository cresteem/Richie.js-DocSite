# Movie & Carousel (Movie, ItemList) 

## What is movie rich result?
Enhance your movie lists with structured data, allowing users to discover movies on Google Search in innovative ways. You can include information such as the movie title, director, and an image. 

<Callout type="info">Movie carousel feature is available exclusively on mobile devices.</Callout>

Kindly Refer [Google Doc](https://developers.google.com/search/docs/appearance/structured-data/movie) to know more about movie rich result.

## What are necessary data for movie rich result?

### 1. Name
```html copy filename="movie.html" /-name/ {3}
<div id="rjs-movie-1">
  <!-- name of the movie -->
  <p class="rjs-movie-1-name">Dawn of Justice</p>
</div>
```

### 2. URL
Richie JS's reserved value on `id` attribute is mandatory for deeplink or movie URL

<Callout type='info'>ID value template is rjs-movie-[number]</Callout>

```html copy filename="movie.html" /id="rjs-movie-1"/
<div id="rjs-movie-1">
  ....
</div>
```

### 3. Thumbnail
```html copy filename="movie.html" /1-img/
<!--images -->
<img class="rjs-movie-1-img" src="https://i.imsur.com/GdoMshp.jpeg" alt="" />
<img class="rjs-movie-1-img" src="https://i.imsur.com/yXDOO5w.png" alt="" />
```

### 4. Release date
```html copy filename="movie.html" /pdt/
<!-- date released YYYY-MM-DD -->
<p class="rjs-movie-1-pdt">2016-02-05</p>
```

### 5. Director
<Callout type="info">More than one director is possible to a entity</Callout>
```html copy filename="movie.html" /-dir/
<!-- directors -->
<p class="rjs-movie-1-dir">Zack Synder</p>
<p class="rjs-movie-1-dir">Another director</p>
```

### 6. Review
```html copy filename="movie.html" /1-reviews/ /<div class="urate">/  /class="rater/ /class="mr"/ /class="rv"/ /class="rpOn"/ /div>/
<!-- user reviews -->
<div class="rjs-movie-1-reviews">

  <!-- user rate instance 1 -->
  <div class="urate">

    <p class="raterP">Darsan</p>
    <!-- rater name|  P(person) and O(organisation) - suffix is reserved-->

    <p class="mr">100</p><!-- max rate range in system -->

    <p class="rv">98</p><!-- ratedValue -->

    <p class="rpOn">IMDB</p><!-- rating published on -->
  
  </div>

  <!-- user rate instance 2 -->
  <div class="urate">

    <p class="raterO">NotIndiaGlitz</p>
    <!-- rater name|  P(person) and O(organisation) - suffix is reserved-->

    <p class="mr">5</p><!-- max rate range in system -->

    <p class="rv">4.5</p><!-- ratedValue -->

    <p class="rpOn">RottenTomato</p><!-- rating published on -->
  
  </div>

</div><!-- user reviews ended -->
```

### 7. Aggregated rating
```html copy filename="movie.html" /1-aggrate/ /arv/ /mr/ /rc/
<!-- aggregate review -->
<div class="rjs-movie-1-aggrate">
  <!-- aggregated rate value-->
  <p class="arv">100</p>

  <!-- max range of rating -->
  <p class="mr">100</p>

  <!-- number of user ratings  -->
  <p class="rc">1000</p>
</div>
```

## Function and parameters
    
    <Callout type='default'>By default input file is overwriiten with result so `destination` is optional</Callout>

    <Callout type='info'>This is only for API method.
    We prefer CLI method for keeping it simpler (refer - [Working with API & CLI](/working-with-richie-js-api-and-cli)).</Callout>

   <Tabs items={['Standard','Carousel']}>
      <Tabs.Tab>
        ```js copy filename="func_params.js" {1,5}
        const richResultType = 'movie';
        const filePath = 'movie.html';
        const destination = 'dist/movie.html'; /* optional */
        
        richie(richResultType, filePath, destination);
        ```
