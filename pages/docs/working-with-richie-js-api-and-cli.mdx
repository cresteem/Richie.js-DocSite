### Set Destination

We can set destination folder using `-d` or `--destDir` flag in `rjs make`.

<Callout type="default">
  By default outputs are stored in `dist` folder.
</Callout>

**Value:** Relative path. <br/>
**Type:** String without quote.

<Tabs items={['npx','yarn']}>
    <Tabs.Tab>
        ```bash copy filename="CMD ~ Bash terminal"
        #store output in different directories
        npx rjs make -d destFolder
        # or
        npx rjs make --destDir destFolder
        ```
    </Tabs.Tab>

    <Tabs.Tab>
        ```bash copy filename="CMD ~ Bash terminal"
        #store output in different directories
        yarn rjs make -d destFolder
        # or
        yarn rjs make --destDir destFolder
        ```
    </Tabs.Tab>

</Tabs>

---

### Omit folders to skip lookup

We can omit folder from lookup list using `--no` or `--omitPatterns`.

**Value:** Relative path or glob pattern. <br/>
**Type:** String without quote.

<Tabs items={['npx','yarn']}>
    <Tabs.Tab>
       ```bash copy filename="CMD ~ Bash terminal"
        #omit files and folders
        npx rjs make --no relativePath/subpath
        # or
        npx rjs make --omitPatterns relativePath/**
        ```
    </Tabs.Tab>

    <Tabs.Tab>
        ```bash copy filename="CMD ~ Bash terminal"
        #omit files and folders
        yarn rjs make --no relativePath/subpath
        # or
        yarn rjs make --omitPatterns relativePath/**
        ```
    </Tabs.Tab>

</Tabs>

### Preserve Previous Files

<Callout type="warning">
  By default Richie JS removes previous outputs / emptying destination
  directory.
</Callout>

If you want to disable this? use preserve flags --norm or -p

<Tabs items={['npx','yarn']}>
    <Tabs.Tab>
      ```bash copy filename="CMD ~ Bash terminal"
      npx rjs make -p
      # or
      npx rjs make --norm
      ```
    </Tabs.Tab>

    <Tabs.Tab>
      ```bash copy filename="CMD ~ Bash terminal"
      yarn rjs make -p
      # or
      yarn rjs make --norm
      ```
    </Tabs.Tab>

</Tabs>

### Unsupported in CLI

Below rich result types are currently not supported using the CLI method. However, you can create them using the API method:

- Site Search Box
- Bread Crumb

## ü§ñ API

Richie JS supports ECMAS and CommonJS syntax for Working with API

<Tabs items={['ECMAS','CommonJS']}>
    <Tabs.Tab>
      ```js filename="demo.cjs" copy {1,16} showLineNumbers
      import { richie } from "@cresteem/richie-js";
      import { mkdirSync } from "fs";
      import { basename, join, relative, dirname } from "path";

      const type = "breadcrumb";
      const inputFile = "pages/breadcrumb/sub-breadcrumb/notindex.html";
      const outputFile = join(
      "./dist",
      relative(process.cwd(), dirname(inputFile)),
      basename(inputFile)
      );

      //making dest folder
      mkdirSync(dirname(outputFile), { recursive: true });

      richie(type, inputFile, outputFile);
      ```
    </Tabs.Tab>

    <Tabs.Tab>
      ```js filename="demo.cjs" copy {1,16} showLineNumbers
      const { richie } = require("@cresteem/richie-js");
      const { mkdirSync } = require("fs");
      const { basename, join, dirname, relative } = require("path");

      const type = "searchbox";
      const inputFile = "pages/Sitesearch/searchpage.html";
      const outputFile = join(
      "./dist",
      relative(process.cwd(), dirname(inputFile)),
      basename(inputFile)
      );

      //making dest folder
      mkdirSync(dirname(outputFile), { recursive: true });

      richie(type, inputFile, outputFile);
      ```
